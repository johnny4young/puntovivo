## Current posture

### Security

- Fastify registers Helmet, explicit CORS allowlists, cookies/JWT, CSRF for
  refresh-cookie unsafe requests, request-scoped logging, and global rate
  limiting.
- Refresh/realtime cookies are httpOnly, SameSite-scoped, and use secure cookies
  only when appropriate for the runtime.
- Renderer access tokens remain in memory; refresh relies on the httpOnly
  cookie path.
- Electron main window keeps `sandbox: true`, `contextIsolation: true`, and
  `nodeIntegration: false`; preload APIs are one-line IPC wrappers.
- Renderer window-open and top-frame navigation are denied by default and route
  allowed external URLs through `shell.openExternal`.
- electron-builder fuses disable Node-as-Electron and CLI inspect/env Node
  options for packaged builds.

### Performance

- Push CI already gates web bundle size, contrast, Lighthouse route vitals, and
  Electron memory budgets.
- The print hardening avoids synchronous `document.write` mutation and reuses
  a single helper with bounded Blob URL lifetime.

### TypeScript / React maintainability

- Web, desktop, and server TypeScript configs already use `strict`,
  `noUncheckedIndexedAccess`, and `exactOptionalPropertyTypes`.
- Server additionally enforces `noImplicitReturns`, `noUnusedLocals`, and
  `noUnusedParameters`.
- Existing `any`/suppression usage is mostly documented around Drizzle
  parametric-table limitations or tests that intentionally bypass types.

### UI/UX and copy

- No user-facing app UI was changed in this branch; live UI smoke is therefore
  not required by AGENTS.md.
- Spanish locale grep found no voseo imperatives/conjugations to fix.
- Common-typo grep produced no actionable typo fixes beyond legitimate words
  such as `mismatched`.

### Permissions and CI

- CI defaults to `contents: read`; PR path filtering narrows job execution.
- Release workflows use broader permissions only where artifacts/pages require
  them, with job-level narrowing for Pages publishing.
- Manual cross-OS desktop validation remains the right place to prove
  platform-specific packaging behavior.

## Proposed follow-ups

1. **Formal security scan** — The Codex Security Deep Security Scan workflow is
   intentionally artifact/worker-driven and was not run as part of this
   implementation PR. Run it separately when a dedicated scan window is
   available.
2. **Trusted Types exploration** — Consider a report-only Trusted Types CSP
   spike for browser-hosted app surfaces once the remaining printable/preview
   HTML paths are centralized.
3. **TS escape-hatch reduction** — Track a small hardening pass to reduce
   non-test `any` usage, prioritizing Drizzle boundary aliases if upstream
   types allow it.
4. **Website stack alignment** — The marketing website remains React 18 /
   Vite 6 while the app is React 19 / Vite 8. This is acceptable because it is
   a separate workspace, but should be planned rather than accidental.
5. **Cross-OS desktop validation** — Before release, run
   `.github/workflows/build-desktop.yml` for macOS and Windows if this branch is
   merged with any future desktop packaging changes.
6. **Existing Lighthouse temp-dir follow-up** — Keep the existing BACKLOG item
   about `scripts/run-lighthouse-gate.mjs` temp-dir cleanup on the next CI/perf
   maintenance pass.
