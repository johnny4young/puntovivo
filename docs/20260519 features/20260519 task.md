# Technical Checklist: Modular POS Development & Verticals

This checklist tracks engineering milestones associated with future development phases of the Puntovivo POS project.

---

## Phase 12: Restaurant and Service Verticals

### 1. `ENG-039g` — Premium Multichannel KDS (Restaurant)
- `[ ]` Design the KDS layout using deep dark background styles and frosted glass cards.
- `[ ]` Implement three-tier color-coded border states based on time elapsed (`isStale`):
  - `0-5 min`: Green neon.
  - `5-10 min`: Amber.
  - `>10 min`: Slow pulsating red breathing glow.
- `[ ]` Create dynamic columns utilizing CSS-Grid to separate tickets by prep station (e.g., Hot Line vs. Bar).
- `[ ]` Hook up Server-Sent Events (SSE) subscriptions to support real-time ticket modifications and cancellations.

### 2. `ENG-042a` — DB Table Schema for Services & Commissions
- `[ ]` Create Drizzle schema migrations for `appointments` and `employee_commissions` tables.
- `[ ]` Set fields for `appointments`: `tenant_id`, `site_id`, `customer_id`, `start_time`, `duration_minutes`, `status`, and `employee_id`.

### 3. `ENG-042b` — Interactive Calendar Scheduling Page
- `[ ]` Create a premium, responsive calendar component on route `/services/appointments` to manage bookings.
- `[ ]` Integrate smooth CSS transition effects when modifying, dragging, or rescheduling calendar timeslots.

### 4. `ENG-042c` — Stylist/Technician Cart Line-item Assignment
- `[ ]` Update `SalesPage.tsx` to support assigning a specific employee to individual line items within the cart.
- `[ ]` Display the assigned employee name cleanly under the item title in the dense cart panel view.

### 5. `ENG-042d` — Commissions calculation in Cash Session Close
- `[ ]` Develop the tRPC commissions router to calculate accrued commission balances for employees.
- `[ ]` Update `CashSessionCloseModal.tsx` to include an employee commission breakdown ready for cash payout at session end.

## Phase 13: Pharmacy Vertical

### 6. `ENG-043a` — Pharmacy Inventory Lots & Expiry Alerts
- `[ ]` Design a database table for inventory lots linked to standard products, tracking expiry timestamps.
- `[ ]` Implement a warning banner in the POS search view highlighting items nearing their expiry thresholds.

### 7. `ENG-043b` — INVIMA Validation & Active Ingredient Generic Lookup
- `[ ]` Add `invima_code` and `active_ingredient` metadata properties to the product catalog model.
- `[ ]` Implement a search recommendation engine in the POS showing cheaper generic alternatives with matching active ingredients.

### 8. `ENG-043c` — Controlled Substance Rx Registration
- `[ ]` Create an interactive modal pop-up during checkout to capture mandatory doctor prescriptions when selling gated medications.

---

## Phase 14: Supermarket Vertical

### 9. `ENG-044a` — Serial Scale Integrations & PLU Barcode Decoders
- `[ ]` Implement a USB/RS232 listener service to pull real-time weights from connected serial weight scales.
- `[ ]` Build a barcode scanner parser to dynamically extract product identification and decimal weights from standard PLU labels.

### 10. `ENG-044b` — Healthy Foods Tax Integration
- `[ ]` Implement the tax calculation engine based on sugar and fat concentrations per SKU.
- `[ ]` Detail healthy foods tax contributions in invoice tax breakdowns and receipt printouts.

---

## Phase 15: Hardware Store Vertical

### 11. `ENG-041a` — Unit Conversions & Cart Conversions
- `[ ]` Build the `product_sale_units` schema mapping fractional units of measure (e.g., converting rolls to linear meters).
- `[ ]` Update the POS line items to allow selecting fractional sale units and dynamically adjust stock counts.

### 12. `ENG-041b` — Internal Barcode Generators
- `[ ]` Write a utility to auto-generate and print EAN-13 barcodes for bulk or unpackaged hardware items.

### 13. `ENG-041c` — Project Templates & Kit Explosions
- `[ ]` Create a template manager allowing admins to pre-bundle standard project kits.
- `[ ]` Integrate a quick-load button in the POS cart to instantly expand and explode a selected project kit into individual editable cart lines.

### 14. `ENG-041d` — High-Density Catalog Searches with SQLite FTS5
- `[ ]` Structure and deploy virtual index tables using SQLite's FTS5 engine.
- `[ ]` Assure query execution response times remain under `100ms` when scanning up to 50,000 SKUs on older POS hardware.
