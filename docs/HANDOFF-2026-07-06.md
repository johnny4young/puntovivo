## Setup local (importante — hay quirks)

- **Node:** `package.json` exige `engines.node >= 24`. Usar **Node 24.x**
  (nvm: `nvm install 24 && nvm use 24`). El entorno cloud traía v22 por defecto
  y hubo que forzar v24 en PATH para verificar.
- **Package manager:** `pnpm@11.4.0` (declarado en `packageManager`). `pnpm install`.
- **Dep bloqueada en cloud:** `@electron/node-gyp` (git dep) fallaba tras el
  proxy; se sorteó reescribiendo el lockfile a `git+https` y con
  `ELECTRON_SKIP_BINARY_DOWNLOAD=1`. **En local con red normal no debería hacer
  falta** — si `pnpm install` se cuelga en esa dep, esa es la causa.
- Para saltar la re-verificación de deps en cada corrida:
  `npm_config_verify_deps_before_run=false`.

## Gotchas aprendidos (leer antes de tocar inventario)

1. **Drizzle interpola `Column` SIN calificar la tabla.** En un `sql` template,
   `${products.id}` renderiza `"id"` a secas. Como `inventory_balances` también
   tiene `id`/`tenant_id`, un subquery correlacionado se ata a la tabla
   equivocada y devuelve 0 en silencio. Por eso `productStockTotalSql` en
   `services/inventory-balances/derive.ts` escribe los identificadores
   **totalmente calificados como SQL crudo**. No "simplificar" eso.
2. **Cada migración que ALTERa una tabla existente necesita un guard de
   adopción** en `db/migration-baseline.ts` (`shouldSeedPostBaselineMigration`)
   para DBs legacy parciales sin la tabla objetivo. Ver el patrón para
   `0003/0004/0005/0007`. Si no, `db.seed.test.ts` falla.
3. **Paridad i18n de error codes.** Un nuevo código de error del server
   (p.ej. `LOT_*`) necesita: clave EN + ES en `errors.server` **y** estar en el
   allowlist de `web/.../translateServerError.ts`. Hay tests de paridad que lo
   exigen (SERVER_ERROR_CODES ↔ i18n ↔ allowlist web).
4. **Seed de stock en tests.** `products.stock` ya no existe: para dar stock
   inicial en un test, insertar una fila en `inventory_balances`
   (`onHand: N` en el sitio activo), no `stock: N` en `products`.
5. **`runFreshSale` es el único punto de débito de stock** (ventas directas y
   creación de draft). El consumo FEFO de lotes vive ahí. Las reversas completas
   (`returnSale`/`voidSale`/`discardDraft`) llaman `restoreLotsForSale`.

## Invariantes que NO se deben romper

- **Snapshot de factores/costos en las líneas** de transacción — cambiar el
  factor de una unidad nunca reescribe historia.
- **`normalizedQuantity`** (unidades base) es la llave de toda la matemática de
  inventario.
- **Migraciones aditivas primero** — nada de ALTER destructivo en tabla caliente
  sin backfill previo (ver `0007` como referencia: backfill → DROP).
